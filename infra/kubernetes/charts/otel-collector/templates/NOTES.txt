OTel Collector DaemonSet deployed as: {{ include "otel-collector.fullname" . }}

REQUIRED: the auth-header Secret "{{ .Values.existingSecret }}" must exist in the
namespace first (keys: otlp-auth-header, loki-auth-header). The chart does NOT
create it.

In this repo the deploy pipeline creates it automatically — .github/workflows/deploy.yml
syncs the Grafana Cloud tokens from GCP Secret Manager into this Secret before the
helm upgrade. See docs/deploy.md → "OTel Collector / Grafana Cloud telemetry".

To create it manually (out-of-band / other clusters):

  kubectl create secret generic {{ .Values.existingSecret }} \
    --from-literal=otlp-auth-header="Basic <base64(instanceId:apiKey)>" \
    --from-literal=loki-auth-header="Basic <base64(instanceId:apiKey)>"
